HYDRA-MSG release artifacts should be reproducible or source-verifiable from a signed tag.

## Current implementation

The source release archive is produced by:

```bash
git archive --format=tar --prefix=hydra-msg-<version>/ HEAD | gzip -n
```

`create-release-package.sh` creates the source archive twice and byte-compares the result before writing the final release directory. It also sets `SOURCE_DATE_EPOCH` from the signed commit timestamp for generated metadata.

## Scope

Current reproducibility guarantee:

```text
source archive: byte-reproducible from the same signed commit
SBOM: reproducible from Cargo.lock, cargo metadata, Python script, and SOURCE_DATE_EPOCH
crate packages: generated by Cargo from the signed source and logged
checksum file: reproducible from the exact artifact set
```

Native binaries and WASM bundles are source-verifiable unless a release explicitly adds stricter byte-for-byte build reproducibility instructions for those artifacts.

## Release package command

```bash
scripts/release/create-release-package.sh vX.Y.Z
```

The script writes:

```text
release-artifacts/vX.Y.Z/
```

and logs Cargo packaging output to:

```text
release-artifacts/vX.Y.Z/logs/cargo-package.log
```

## Verifier workflow

```bash
git fetch --tags
git verify-tag vX.Y.Z
git checkout vX.Y.Z
scripts/release/create-release-package.sh vX.Y.Z
scripts/release/verify-release-artifacts.sh vX.Y.Z
```

For a pure independent rebuild, compare the regenerated source archive and SBOM hashes against the published `SHA256SUMS.txt`.

## Environment record

Each release evidence directory should record:

```text
OS and architecture
rustc --version --verbose
cargo --version --verbose
python3 --version
git --version
gpg --version
sha256sum --version
SOURCE_DATE_EPOCH
```
