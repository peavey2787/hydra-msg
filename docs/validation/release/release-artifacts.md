## Current implementation

The release scripts create a versioned folder:

```text
release-artifacts/<version>/
```

Primary script:

```bash
scripts/release/create-release-package.sh vX.Y.Z
```

It creates:

```text
hydra-msg-vX.Y.Z-source.tar.gz
crates/*.crate
sbom/hydra-msg-vX.Y.Z-cyclonedx.json
sbom/hydra-msg-vX.Y.Z-cargo-metadata.json
RELEASE-MANIFEST.txt
SHA256SUMS.txt
logs/cargo-package.log
```

The script creates the source archive twice and compares the bytes before publishing the archive into the release directory. The source archive is generated with `git archive` and `gzip -n` from the selected commit.

## Artifact set

A production release should publish the following artifacts when applicable:

| Artifact | Required? | Notes |
|---|---:|---|
| Source archive from signed tag | Yes | Created by `scripts/release/create-release-package.sh`. |
| `Cargo.lock` | Yes | Included in the source archive and identified in release evidence. |
| CycloneDX SBOM | Yes | Generated from `cargo metadata --locked`. |
| Cargo metadata JSON | Yes | Raw dependency evidence used to generate and audit the SBOM. |
| `SHA256SUMS.txt` | Yes | SHA-256 hashes for every published artifact. |
| `SHA256SUMS.txt.asc` | Yes | Detached GPG signature over the checksum file. |
| `.crate` package artifacts | If publishing crates | Generated by `cargo package --workspace --locked`. |
| WASM web package | If releasing browser SDK artifacts | Build from the signed source and document the exact command. |
| Release evidence summary | Yes | Human-readable index of validation logs and reports. |

Examples and QA-only packages are not production runtime artifacts unless a release explicitly says otherwise.

## Hash publication

Every published artifact must appear in the checksum file:

```text
SHA256SUMS.txt
SHA256SUMS.txt.asc
```

Verify with:

```bash
gpg --verify SHA256SUMS.txt.asc SHA256SUMS.txt
sha256sum -c SHA256SUMS.txt
```

## Artifact naming

Use stable, versioned names:

```text
hydra-msg-<version>-source.tar.gz
hydra-msg-<version>-cyclonedx.json
hydra-msg-<version>-cargo-metadata.json
hydra-msg-<version>-wasm-web.tar.gz
hydra-msg-<version>-release-evidence.tar.gz
SHA256SUMS.txt
SHA256SUMS.txt.asc
```

Do not publish anonymous build outputs such as `dist.zip`, `target.zip`, or unsigned ad hoc archives as release artifacts.

## Verification commands for release notes

Release notes should include commands shaped like:

```bash
git verify-tag vX.Y.Z
gpg --verify SHA256SUMS.txt.asc SHA256SUMS.txt
sha256sum -c SHA256SUMS.txt
```

If the project later uses Sigstore/cosign instead of or in addition to GPG, update this document and [Release signing](release-signing.md) with exact verification commands.
