Normal settings

No required settings are needed for normal encrypted chat. Create or unlock an identity, trust a contact, then start chatting.

  • Identity passwords are never stored.
  • Private keys remain encrypted at rest.
  • The local browser GUI remains loopback-only unless started with an explicit dangerous CLI flag.
Advanced message intake and chat-start policy

These controls decide how reviewed inbound messages and new group shells are filtered locally. They do not enable a production remote relay or mailbox.

Advanced rekey policy

Rekey thresholds control when app sessions/groups should rotate derived session or group state. Lower values rotate more often; higher values reduce churn. These settings do not change HYDRA wire formats or cryptographic constants.

Advanced local storage directory

The current data directory is shown for transparency, but changing it from the browser UI is intentionally disabled. Choose a different local app-state root before startup with HYDRA_APP_DATA_DIR or a documented CLI workflow.
